# 🔒 Project Cyber Security (კრიპტოგრაფიული მეთოდების ანალიზი)

ეს პროექტი წარმოადგენს სამი სხვადასხვა კრიპტოგრაფიული მეთოდის - **კეისრის შიფრის (Caesar)**, **XOR შიფრის** და **AES შიფრის** - იმპლემენტაციასა და ანალიზს. პროექტის მიზანია, თვალსაჩინო გახადოს მარტივი (ისტორიული) და თანამედროვე (უსაფრთხო) დაშიფვრის პრინციპებს შორის არსებული განსხვავება.

---

## ⚙️ დაშიფვრის მეთოდები და მათი ლოგიკა

პროექტში გამოყენებულია სამივე მეთოდი, რათა ტექსტი გადავიდეს დაშიფრულ ფორმაში, თუმცა მათი მუშაობის სირთულე და უსაფრთხოება რადიკალურად განსხვავდება.

### 1. 🔑 Caesar Cipher (კეისრის შიფრი) - (`caesar.py`)

**კეისრის შიფრი ყველაზე მარტივი მეთოდია**, რომელსაც ამჟამად უსაფრთხოებისთვის არ იყენებენ.

* **პრინციპი:** თითოეული ასო ანაცვლებს წინ, მოცემული რიცხვით (`shift`).
* **უსაფრთხოება:** დაბალი (ადვილად გატეხადი).

**რა ხდება კოდში:**
1.  **ნულამდე დაყვანა:** ასოს ASCII კოდიდან გამოაკლდება ანბანის დასაწყისის კოდი (A ან a), რათა თვლა დაიწყოს ნულიდან (0-დან 25-მდე).
2.  **გადაადგილება და შემოხვევა:** ნულოვან ინდექსს ემატება `shift` რიცხვი. გამოიყენება **`% 26` (მოდულური ოპერაცია)**, რაც უზრუნველყოფს, რომ ანბანის ბოლოდან (`Z`) თვლა თავიდან დაიწყოს (`A`-ზე).

### 2. 🛡️ XOR შიფრი - (`xor.py`)

XOR შიფრი იყენებს **ბინარულ ლოგიკურ ოპერაციას** (`^`), რათა ტექსტი და გასაღები ერთმანეთში დამალოს.

* **პრინციპი:** თითოეული სიმბოლოს რიცხვითი კოდი (ASCII) უკეთდება **XOR ოპერაციას** გასაღებთან (`key`).
* **უსაფრთხოება:** დაბალი (არასაიმედოა მოკლე, განმეორებადი გასაღებების დროს).

**მთავარი ლოგიკა:** XOR ოპერაციის თვისებაა, რომ ერთი და იგივე ფუნქცია და გასაღები გამოიყენება როგორც დაშიფვრისთვის, ისე გაშიფვრისთვის.

### 3. 🔐 AES შიფრი (Advanced Encryption Standard) - (`aes.py`)

AES არის **თანამედროვე, სიმეტრიული ბლოკური შიფრი**, რომელიც დღეს გლობალური ინდუსტრიული სტანდარტია.

* **პრინციპი:** მონაცემები მუშავდება 128-ბიტიან ბლოკებად და გადის მრავალ რთულ რაუნდს, მათ შორის ჩანაცვლებას, გადაადგილებას და შერევას.
* **უსაფრთხოება:** **უმაღლესი დონე**. გამოიყენება HTTPS, VPN, Wi-Fi (WPA2/3) და მყარი დისკების დაშიფვრისთვის.

**გამოყენების კრიტიკული ასპექტები:**
* **სპეციალური ბიბლიოთეკა:** AES-ის სირთულის გამო, კოდში გამოიყენება უსაფრთხო ბიბლიოთეკები (`PyCryptodome` ან `Cryptography`).
* **Padding (შევსება):** ტექსტი უნდა შეივსოს, რათა მისი ზომა იყოს 16 ბაიტის ზუსტი ჯერადი.
* **IV (Initialization Vector):** გამოიყენება შემთხვევითი ვექტორი, რათა უზრუნველყოფილი იყოს, რომ ერთი და იგივე ტექსტის ყოველი დაშიფვრა განსხვავებული იყოს.
